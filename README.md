#### Git Branch Strategy

- `main`
    - 배포 브랜치
    - 직접 커밋 금지
    - dev 브랜치에서 머지만 허용 (Create a merge commit)
- `dev`
    - 메인 개발 브랜치
    - 직접 커밋 금지
    - 하위 브랜치에서 머지만 허용 (Rebase & merge)
    - 모든 개발 작업은 dev 브랜치를 기준으로 개별 브랜치를 생성하여 진행
